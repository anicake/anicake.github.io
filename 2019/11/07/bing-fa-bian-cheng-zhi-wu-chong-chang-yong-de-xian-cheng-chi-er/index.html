并发编程之五种常见的线程池

前言

在日常的开发中往往不需要我们自己实现一个线程池，因为java api中为我们提供了5种封装好的线程池，但是我们选择的时候需要明白每种线程池具体的使用场景和其中的原理，不能盲目的使用线程池，不然可能会适得其反，下面我们就来了解下java api中提供的线程池的用法：

五种线程池的分类和作用

1、newCachedThreadPool

public static ExecutorService newCachedThreadPool() {
    return new ThreadPoolExecutor(0, Integer.MAX_VALUE,
                                  60L, TimeUnit.SECONDS,
                                  new SynchronousQueue<Runnable>());
}

newCachedThreadPool是一个可缓存的无限线程池，如果线程池的长度过长需要处理，可灵活的回收线程。
特征
(1)线程池中的数量没有固定，默认可达最大值Interge.MAX_VALUE。
(2)线程池中的线程会缓存起来复用，保存时间默认为60s
(3)当线程池中没有可用线程时，会新创建一个线程
测试用例：

ExecutorService pool =  Executors.newCachedThreadPool();
pool.submit(()->{
    for(int i=0;i<10;i++){
        System.out.println(Thread.currentThread().getName()+":"+i);
    }
});
pool.shutdown();

测试结果

pool-1-thread-1:1
pool-1-thread-1:2
pool-1-thread-1:3
pool-1-thread-1:4
pool-1-thread-1:5
pool-1-thread-1:6
pool-1-thread-1:7
pool-1-thread-1:8
pool-1-thread-1:9
适用场景：只有在使用无界线程池或者有饱和策略时才建议使用该队列。。

2、newFixedThreadPool

public static ExecutorService newFixedThreadPool(int nThreads) {
    return new ThreadPoolExecutor(nThreads, nThreads,
                                  0L, TimeUnit.MILLISECONDS,
                                  new LinkedBlockingQueue<Runnable>());
}

newFixedThreadPool创建一个可重用的固定线程数的线程池，且有一个共享的无界阻塞队列。在大多数时候，nThreads线程都在处理任务活动的状态。如果在nThreads线程都在活动的情况下，有新提交的任务，那么会将新提交的任务放到LinkedBlockingQueue中等待，等到nThreads线程中有执行完任务的线程释放后执行。在线程被显示关闭之前，线程池会一直存在这个线程。
特征
(1)线程池中的线程数处于一定的数量，可以很好地控制线程的并发量
(2)线程可以被重复使用，线程在关闭之前都一直存在
(3)当线程池中线程的数量已经达到核心线程数时，超出的任务会加入到队列中去
测试用例

ExecutorService executor = Executors.newFixedThreadPool(5);
for (int i = 0;i<10;i++) {
    int finalI = i;
    executor.execute(()->{
        System.out.println(Thread.currentThread().getName()+":"+finalI);
    });
}
executor.shutdown();

测试结果
pool-1-thread-1:0
pool-1-thread-1:5
pool-1-thread-1:6
pool-1-thread-1:7
pool-1-thread-1:8
pool-1-thread-2:1
pool-1-thread-1:9
pool-1-thread-4:3
pool-1-thread-3:2
pool-1-thread-5:4
从上面的输出结果可以看出，线程池只创建了5个线程，且线程1被重复使用多次
适用场景：程序的并发量很高不可控，需要避免生成过多线程。（注意当使用无界队列时，队列中积压的任务数量如果太多可能会导致系统OOM）

3、newSingleThreadExecutor

public static ExecutorService newSingleThreadExecutor() {
    return new FinalizableDelegatedExecutorService
        (new ThreadPoolExecutor(1, 1,
                                0L, TimeUnit.MILLISECONDS,
                                new LinkedBlockingQueue<Runnable>()));
}

newSingleThreadExecutor创建一个每次只会有一个任务在执行的线程池，新添加的任务需要到阻塞队列中等待。
特征
(1)线程池中只会有一个任务在执行
(2)线程的执行会按照任务的提交顺序执行
测试用例

ExecutorService pool = Executors.newSingleThreadExecutor();
for (int i = 0 ; i < 10;i++) {
    final int finalI = i;
    pool.submit(()->{
        System.out.println(Thread.currentThread().getName()+":"+finalI);
    });
}
pool.shutdown();

测试结果
pool-1-thread-1:0
pool-1-thread-1:1
pool-1-thread-1:2
pool-1-thread-1:3
pool-1-thread-1:4
pool-1-thread-1:5
pool-1-thread-1:6
pool-1-thread-1:7
pool-1-thread-1:8
pool-1-thread-1:9
适用场景：线程池的任务需要按顺序执行的程序

4、newScheduledThreadPool

public static ScheduledExecutorService newScheduledThreadPool(int corePoolSize) {
    return new ScheduledThreadPoolExecutor(corePoolSize);
}
public ScheduledThreadPoolExecutor(int corePoolSize) {
    super(corePoolSize, Integer.MAX_VALUE, 0, NANOSECONDS,
          new DelayedWorkQueue());
}

newScheduledThreadPool创建一个核心线程数定长的线程池，并且可以延时运行或者周期性的运行
特征
(1)线程池中的具有指定的线程数量
(2)可以周期性或延时运行线程池中的任务
测试用例
ScheduledExecutorService pool = Executors.newScheduledThreadPool(1);
long start = System.currentTimeMillis();
Runnable runnable = new Runnable() {
@Override
public void run() {
try {
Thread.sleep(1000);
} catch (InterruptedException e) {
e.printStackTrace();
}
System.out.println(Thread.currentThread().getName() + “Scheduled” + “:”+ (System.currentTimeMillis()-start)/1000);
}
};
// pool.scheduleAtFixedRate(runnable,2,3,TimeUnit.SECONDS); 程序启动后2秒才开始执行，接着每3s执行一次 不可以加shutdown();
// pool.scheduleWithFixedDelay(runnable,2,3,TimeUnit.SECONDS); 程序启动后2s开始执行，之后在前一次执行完后3s执行一次不可以加shutdown();
pool.schedule(runnable,3,TimeUnit.SECONDS);//程序启动3s后执行
pool.shutdown();

测试结果

pool-1-thread-1Scheduled:6
pool-1-thread-1Scheduled:9
pool-1-thread-1Scheduled:12
pool-1-thread-1Scheduled:15
scheduleWithFixedDelay:
pool-1-thread-1Scheduled:3
pool-1-thread-1Scheduled:7
pool-1-thread-1Scheduled:11
pool-1-thread-1Scheduled:15
pool-1-thread-1Scheduled:19
schedule:
pool-1-thread-1Scheduled:3
适用场景：业务需要线程池中的任务延时或按照一定的固定的频率计划执行

5、newSingleThreadScheduledExecutor
----------------------------
---
```bash
public static ScheduledExecutorService newSingleThreadScheduledExecutor() {
    return new DelegatedScheduledExecutorService
        (new ScheduledThreadPoolExecutor(1));
}

newSingleThreadScheduledExecutor也是一个调度线程池工作原理和newScheduledThreadPool一样，不过线程池中只有一个线程在工作，其余任务在队列中等待

如何合理配置线程池

线程池中的线程数并不是越多越好，合理的配置线程池的能更好的提高系统的性能，配置线程池可以从下面几个方面入手：

  • **(1)、任务的性质：判断任务是IO密集型还是CPU密集型
  • **(2)、任务的优先级
  • **(3)、任务的执行时间
    性质不同的任务可以用不同规模的线程池去处理:
  • **CPU密集型的任务，配置线程池的线程数尽可能的小一点，一般可以配置为(N+1)个。在这种情况下，创建过多的线程反而不好，因为多个任务在处理的状态，处理器核心需要在不同的线程频繁的切换上下文，这样切换是对系统的性能很大的消耗。N处理器CPU的核数
  • **IO密集型的任务由于CPU利用率不高，我们需要设置更多线程数来提高性能。一般可以配置为(2N+1)个。因为当IO密集型的任务在执行的时候，线程会被阻塞，此时处理器核心可以立即切换上下文以便处理其他就绪线程，如果只配置了处理器核心数的线程数时，就算有其他待处理的任务，我们也拿不出更多的线程去执行了。
    优先级不同的任务我们可以使用PriorityBlockingQueue优先队列，它可以让优先级更高的任务先执行
    任务的执行时间不同的任务我们可以用不同规模的线程池来执行，也可以用具有优先级的线程池去优先执行执行时间短的任务
